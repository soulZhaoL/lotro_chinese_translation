# 变更提案: 文档/Mock/布局/测试与联通方案

## 需求背景
当前项目已具备后端骨架、前端骨架与基础接口，但缺少面向使用者的说明文档、前端 Mock 能力、标准布局与可执行的联通步骤，同时后端接口注释与测试覆盖不足，影响本地开发与 MVP 验证效率。

## 变更内容
1. 新增前后端使用说明文档（含依赖安装、环境配置、启动方式）
2. 后端接口补充必要代码注释，并输出 API 文档（Markdown + FastAPI 自带文档）
3. 前端集成 Vite mock 插件，支持无后端情况下的自测
4. 前端布局调整为“顶部导航 + 左侧二级导航”
5. 后端单元测试完善（仅后端）
6. 制定前后端联通步骤，指导完成 MVP

## 影响范围
- **模块:** 前端 UI、后端 API、测试、文档
- **文件:** docs/*, server/routes/*, web/*, tests/*, requirements.txt, web/package.json
- **API:** 所有已有接口
- **数据:** 无新增

## 核心场景

### 需求: 使用说明文档
**模块:** 文档
提供前端与后端的环境依赖、配置、启动与联通流程。

#### 场景: 新同事本地启动
- 按文档完成环境配置与服务启动
- 可以跑通 Mock 或联通后端

### 需求: 后端接口注释与 API 文档
**模块:** 后端 API
补充必要注释并提供 Markdown API 文档。

#### 场景: 快速理解接口
- 通过注释与文档理解接口入参/出参

### 需求: 前端 Mock
**模块:** 前端
支持本地 Mock，便于脱离后端自测。

#### 场景: 前端独立开发
- 开启 Mock 后可进行列表/翻译/词典的页面交互测试

### 需求: 页面布局
**模块:** 前端 UI
采用顶部导航 + 左侧二级导航结构。

#### 场景: 页面导航
- 顶部显示系统级导航
- 左侧显示模块级入口

### 需求: 后端测试
**模块:** 测试
仅覆盖后端关键接口/逻辑。

#### 场景: 小范围自测
- 可运行 pytest 完成接口核心路径验证

### 需求: 联通方案
**模块:** 文档
提供可执行步骤，指导完成 MVP 联通。

#### 场景: 前后端联通
- 按步骤完成后端启动与前端连接

## 风险评估
- **风险:** Mock 与真实 API 行为不一致
- **缓解:** 文档明确 Mock 覆盖范围，关键路径用后端联通验证
