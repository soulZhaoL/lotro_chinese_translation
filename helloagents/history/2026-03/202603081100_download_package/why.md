# 变更提案：下载汉化包功能

## 背景
翻译人员完成汉化后，需要一种方式将数据库中的翻译结果导出为游戏可直接使用的格式。现有"导出"功能按 part 分段导出（供翻译人员查看），无法还原游戏原始的多段合并格式。

## 目标
新增"下载汉化包"功能，按 fid 分组合并所有 part，还原 `|||` 分隔的原始协议格式，生成游戏可直接使用的 Excel 文件。

## 需求确认
- textId 协议格式：严格还原 `textId:::::::[text]` 格式（6冒号）
- part_range：范围压缩（如 `1-3`, `1-1`）
- 文件名：固定 `text_work.xlsx`
- 空译文处理：取原文填充
- 筛选支持：与"导出"功能相同的所有筛选条件（fid/status/sourceKeyword 等）
- 数据规模：约 80 万行，未来可能超 100 万，必须避免内存溢出

## 成功标准
- [ ] 后端 API 正常返回 Excel 文件
- [ ] 同一 fid 的多个 part 按 `|||` 正确合并，还原协议格式
- [ ] 空译文自动取原文
- [ ] 文件名为 text_work.xlsx
- [ ] 80 万行数据导出不崩溃（流式处理）
